## Introduction

### What is Hyprland?

Hyprland is a dynamic tiling Wayland compositor written in C++. It's designed to be highly customizable, feature-rich, and visually appealing while maintaining excellent performance. Hyprland provides advanced window management capabilities, smooth animations, and extensive configuration options that allow users to create a truly personalized desktop experience.

### Why Hyprland?

Hyprland is an excellent choice for users who want a modern, efficient, and highly customizable desktop environment. As a Wayland compositor, it offers better security, performance, and multi-monitor support compared to traditional X11 window managers. Hyprland's tiling window management automatically organizes your windows for optimal screen space usage, boosting productivity.

The compositor is particularly appealing for power users and developers who appreciate fine-grained control over their desktop environment. With its extensive configuration options, smooth animations, and active development community, Hyprland represents the cutting edge of Linux desktop technology while remaining stable and reliable for daily use.

## Installation

### Requirements

- A usb drive with at least 8GB of storage
- A ISO image of Arch Linux [Download](https://archlinux.org/download/)
- A computer with a 64-bit processor
- A stable internet connection

### Installation steps

#### 1. Setup

**First**, you need to download the Arch Linux ISO image from the Arch Linux website. You can download the ISO image from the Arch Linux website by clicking on the download link on the Arch Linux website.

**Second**, use `Ventoy` to burn the ISO image to the USB drive. You can download Ventoy from this [website](https://www.ventoy.net/en/download.html). Once you have downloaded Ventoy, you can craete bootable USB, then copy the ISO file to this.

**Third**, plug the USB drive into the computer that you want to install Arch Linux on. Then, you need to boot the computer from the USB drive. You can do this by restarting the computer and pressing the boot menu key when the computer starts up. The boot menu key is usually F12, but it can vary depending on the computer. Once you have booted the computer from the USB drive, you need to select the Arch Linux installation option from the boot menu.

**Finally**, started to install Arch Linux on your computer. You can follow the Arch Linux installation guide to install Arch Linux on your computer.

# Increase the font size

#### 2. Partition the disk with `btrfs` filesystem

```bash
# Show the available disk
lsblk

# Choose the drive has at least 50GB of storage
# Use cfdisk to create a new partition table on the drive
cfdisk /dev/[drive_name]
```

Create the partitions on the drive.

- Move the cursor to the free space and press `Enter`
- Select the `New` option and press `Enter`
- Set the size of the partition is `512M` and press `Enter`
- Select `type` to `EFI System` and press `Enter`
- Go to the free space again and select the `New` option and press `Enter`
- Set the size of the partition is at least `40GB` and press `Enter`

  <Callout>

  This partition is used to home directory

  You can set the size according to your needs. If you have more storage, you can set a larger size for this partition. Completed system installed only takes arrowd 6GB of storage, but you may want to have more space for your files and applications.

  </Callout>

- Select `write` and press `Enter` to write the changes to the disk then `quit` to exit the cfdisk

```bash
# Format the partitions
mkfs.fat -F32 /dev/nvme0n1p1 # Replace /dev/nvme0n1p1 with your EFI (512M) partition
mkfs.btrfs /dev/nvme0n1p2    # Replace /dev/nvme0n1p2 with your Home (>= 40G) partition

# Mount the partitions to the system
mount --mkdir /dev/nvme0n1p2 /mnt

# Create the subvolumes for the system (it based on the https://wiki.cachyos.org/installation/filesystem/#subvolume-layout)
btrfs subvolume create /mnt/@
btrfs subvolume create /mnt/@home
btrfs subvolume create /mnt/@root
btrfs subvolume create /mnt/@srv
btrfs subvolume create /mnt/@cache
btrfs subvolume create /mnt/@tmp
btrfs subvolume create /mnt/@log

# Unmount the root fs
umount /mnt

# Mount the subvolumes to the system with zstd compression
mount -o compress=zstd,subvol=@ /dev/nvme0n1p2 /mnt
mkdir -p /mnt/{home,root,srv,var/cache,var/tmp,var/log}
mount -o compress=zstd,subvol=@home /dev/nvme0n1p2 /mnt/home
mount -o compress=zstd,subvol=@root /dev/nvme0n1p2 /mnt/root
mount -o compress=zstd,subvol=@srv /dev/nvme0n1p2 /mnt/srv
mount -o compress=zstd,subvol=@cache /dev/nvme0n1p2 /mnt/var/cache
mount -o compress=zstd,subvol=@tmp /dev/nvme0n1p2 /mnt/var/tmp
mount -o compress=zstd,subvol=@log /dev/nvme0n1p2 /mnt/var/log

# Mount the EFI partition
mkdir -p /mnt/efi
mount /dev/nvme0n1p1 /mnt/efi
```

#### 3. Install packages

- `base`, `base-devel`, `linux`, and `linux-firmware` are the essential packages for the Arch Linux system.
- `intel-ucode` for Intel CPU microcode updates, which can improve system stability and security. (use `amd-ucode` if you have AMD CPU)
- `btrfs-assistant` and `btrfs-progs` are the tools for managing the Btrfs filesystem.
- `efibootmgr`, `grub`, `grub-btrfs`, and `os-prober` are the tools for managing the bootloader and dual booting with Windows
- `inotify-tools` is the tool for monitoring file system events, which is used by `snapper` to automatically create snapshots when there are changes in the file system.
- `snap-pac` and `snapper` are the tools for managing Btrfs snapshots, which can be used to create and manage snapshots of the file system.
- `pipewire`, `pipewire-alsa`, `pipewire-pulse`, and `pipewire-jack` are the tools for managing audio on the system, which is used for audio management
- `git` is the version control system, which is used for managing the configuration files and dotfiles.
- `neovim` is the text editor, which is used for editing the configuration files and writing code.
- `networkmanager` is the tool for managing network connections, which is used for managing the network connections on the system.
- `openssh` is the tool for managing SSH connections, which is used for managing SSH connections on the system.
- `reflector` is the tool for managing the mirror list, which is used for managing the mirror list on the system.
- `sudo` is the tool for managing the sudo permissions, which is used for managing the sudo permissions on the system.
- `wireplumber` is the tool for managing the audio routing, which is used for managing the audio routing on the system.

```bash
pacstrap -K /mnt base base-devel linux linux-firmware intel-ucode \
  btrfs-assistant btrfs-progs efibootmgr grub grub-btrfs inotify-tools os-prober snap-pac snapper \
  pipewire pipewire-alsa pipewire-pulse pipewire-jack \
  git neovim networkmanager openssh reflector sudo wireplumber
```

<Callout type="warning">

If you encounter a "Failed to install packages to new root" error when running `archinstall`, you may need to update the Arch Linux keyring before proceeding:

```bash
pacman -Sy archlinux-keyring
```

</Callout>

#### 4. Fstab and chroot

```bash
genfstab -U /mnt >> /mnt/etc/fstab

# verify the fstab file
cat /mnt/etc/fstab

# Chroot into the new system
arch-chroot /mnt
```

#### 5. Setup timezone, language and keyboard layout

Set the timezone for your system by creating a symbolic link from the appropriate timezone file in the `/usr/share/zoneinfo` directory to the `/etc/localtime` file.

```bash
# Replace Asia/Ho_Chi_Minh with your timezone
ln -sf /usr/share/zoneinfo/Asia/Ho_Chi_Minh /etc/localtime

# Now synchronize the hardware clock with the system clock by running the following command.
hwclock --systohc
```

Uncomment the locale you want to use in the `/etc/locale.gen` file and generate the locale by running the following command.

```bash
nvim /etc/locale.gen

locale-gen
```

Next, set the `LANG` variable in the `/etc/locale.conf` file to the locale you want to use. For example, if you want to use `en_US.UTF-8` locale, you need to set the `LANG` variable as follows.

```bash title="/etc/locale.conf"
LANG=en_US.UTF-8"
```

Finally, set the keyboard layout for the system. For example, if you want to use `us` keyboard layout, you can run the following command.

```bash title="/etc/vconsole.conf"
KEYMAP=us
```

#### 6. Hostname and Host configuration

Set the hostname for your system by creating a file named `/etc/hostname` and adding your desired hostname to it.

```bash title="/etc/hostname"
arch-btw
```

Next, you need to configure the hosts file to map the hostname to the loopback address. You can do this by editing the `/etc/hosts` file and adding the following lines.

```bash title="/etc/hosts"
127.0.0.1 localhost
::1 localhost
127.0.1.1 arch-btw
```

#### 7. Root password and user creation

Set the root password for your system by running the following command and following the prompts.

```bash
passwd
```

Next, create a new user for your system by running the following command and following the prompts. Replace `username` with your desired username.

```bash
useradd -m -G wheel username
passwd username
```

Finally, you need to edit the sudoers file to allow the new user to use `sudo` command. You can do this by running the following command and finding the line that is `## Uncomment to allow members of group wheel to execute any command` and uncommenting the line below it.

```bash
EDITOR=neovim visudo
```

```bash title="/etc/sudoers"
...
## Uncomment to allow members of group wheel to execute any command
%wheel ALL=(ALL:ALL) ALL
```

#### 8. Install and configure the bootloader (dual boot with Windows 11)

```bash
grub-install --target=x86_64-efi --efi-directory=/efi --bootloader-id=arch
grub-mkconfig -o /boot/grub/grub.cfg
```

Then, config `grub` boot menu

```bash
nvim /etc/default/grub
```

Uncomment the following line

```bash title="/etc/default/grub"
GRUB_DISABLE_OS_PROBER=false
```

Generate the grub configuration file again to apply the changes

```bash
# Mount the EFI partitions of Windows 11
lsblk # Check the EFI partition
mkdir /mnt/windows
mount /dev/[drive_name] /mnt/windows # Replace [drive_name] with the EFI partition of Windows 11

# Run os-prober to detect the Windows 11 installation and add it to the grub configuration
sudo os-prober
sudo grub-mkconfig -o /boot/grub/grub.cfg
```

#### 9. Umount everything and reboot

```bash
# Enable the NetworkManager service to start at boot
systemctl enable NetworkManager

# Exit the chroot environment
exit

# Unmount everything to check if the drive is busy
umount -R /mnt

# Reboot the system and unplug the installation media
reboot
```

### 10. Setup `snapper` for Btrfs snapshots

Initialize a snapshot configuration named `root` for the root (/) filesystem:

```bash
snapper -c root create-config /
```

Edit the service file to include the `--syslog` flag for proper logging, ensuring snapshots are dynamically added to the GRUB boot menu:

```bash
sudo systemctl edit --full grub-btrfsd

# Enable the grub-btrfsd service by adding the following lines to the file
sudo systemctl enable grub-btrfsd
```

### Manual installation

If you prefer to inspect what goes onto your system, want to customize the setup, or need to troubleshoot a specific step, you can follow the step-by-step breakdown below.

#### 1. Install necessary packages

1.1. Install `yay`

```bash
pacman -Syu --needed git base-devel
git clone https://aur.archlinux.org/yay.git ~/yay
cd ~/yay
makepkg -si
rm -rf ~/yay
```

1.2. Clone `dotfiles` repository to get the list of packages to install

```bash
git clone https://github.com/tiesen243/dotfiles.git ~/dotfiles
```

1.3. Install packages from `package.txt`

```bash
yay -S --needed --noconfirm --answerclean All --answerdiff --None $(grep -v '^#' ~/dotfiles/package.txt)
```

1.4. Set `zsh` as the default shell

```bash
chsh -s $(which zsh)
```

#### 2. Apply configuration

2.1. Clone `dotfiles`

```bash
git clone git@github.com:tiesen243/dotfiles.git ~/dotfiles
```

2.2. Remove existed config and create the symbolic links to the config files

```bash
rm -rf ~/{.cache,.local,.zshrc}
rm -rf ~/.config/{Thunar,btop,fastfetch,git,gtk-3.0,gtk-4.0,hypr,kitty,lazygit,lsd,matugen,nvim,quickshell,zsh}
ln -s ~/dotfiles/{Thunar,btop,fastfetch,git,gtk-3.0,gtk-4.0,hypr,kitty,lazygit,lsd,matugen,nvim,quickshell,zsh} ~/.config

# For who like lowercase stuffs
rm -rf ~/{Desktop,Documents,Downloads,Music,Pictures,Projects,Public,Templates,Videos} && mkdir -p ~/{documents,downloads,pictures,projects,videos}
cat <<EOF > "$HOME/.config/user-dirs.dirs"
XDG_DESKTOP_DIR="\$HOME/"
XDG_DOWNLOAD_DIR="\$HOME/downloads"
XDG_TEMPLATES_DIR="\$HOME/"
XDG_PUBLICSHARE_DIR="\$HOME/"
XDG_DOCUMENTS_DIR="\$HOME/documents"
XDG_MUSIC_DIR="\$HOME/"
XDG_PICTURES_DIR="\$HOME/pictures"
XDG_VIDEOS_DIR="\$HOME/videos"
XDG_PROJECTS_DIR="\$HOME/projects"
EOF
xdg-user-dirs-update
```

2.3. Generate default theme

```bash
matugen color hex 3f5ec2
```

2.4. Make all scripts in the `dotfiles/scripts` directory executable:

```bash
sudo chmod +x ~/dotfiles/scripts/*
```

#### 4. Fix windows 11 time

If it show wrong time on Windows, you can run the following command to fix it

```bash
timedatectl set-local-rtc 1
```

Or edit Registry Editor in Windows and change the value of `RealTimeIsUniversal` to `1` (DWORD) in the following path `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\TimeZoneInformation`

#### 5. Install GPU drivers

To have better performance and support for your graphics card, you need to install the appropriate GPU drivers for your system. The installation process varies depending on the type of graphics card you have.

1. For `Nvidia` users

   If you are using `Nvidia` graphics card, you need to install `nvidia` and `nvidia-utils` packages and add the following line to your `/etc/mkinitcpio.conf` file.

   ```bash
   sudo pacman -S nvidia-open-dkms nvidia-utils linux-headers # or linux-cachyos-headers if you are using cachyos kernel
   ```

   Then, regenerate the initramfs file to apply the changes. (skip this step if you are using `linux-cachyos` kernel, because it already includes `nvidia` module in the initramfs)

   ```bash title="/etc/mkinitcpio.conf"
   MODULES=(... i915 nvidia nvidia_modeset nvidia_uvm nvidia_drm)
   ```

   ```bash
   sudo mkinitcpio -P
   ```

   Then, add the following line to your `/etc/default/grub` file to enable `nvidia` module in the grub.

   ```bash title="/etc/default/grub"
   GRUB_CMDLINE_LINUX_DEFAULT="... nvidia-drm.modeset=1 nvidia-drm.fbdev=1"
   ```

   Finally, regenerate the grub configuration file to apply the changes.

   ```bash
   sudo grub-mkconfig -o /boot/grub/grub.cfg
   ```

2. For `AMD` users

   If you are using `AMD` graphics card, you need to install `xf86-video-amdgpu` package.

   ```bash
   sudo pacman -S mesa vulkan-radeon libva-mesa-driver mesa-vdpau
   ```

#### 6. Add Cachyos's Optimized Repositories (optional)

CachyOS provides optimized repositories for Arch Linux that can improve the performance of your system. To add Cachyos's optimized repositories, you can follow the steps below:

```bash
curl https://mirror.cachyos.org/cachyos-repo.tar.xz -o cachyos-repo.tar.xz
tar xvf cachyos-repo.tar.xz && cd cachyos-repo
sudo ./cachyos-repo.sh
```

Install Cachyos kernels (optional):

```bash
sudo pacman -S linux-cachyos linux-cachyos-headers
sudo grub-mkconfig -o /boot/grub/grub.cfg
```

#### 7. Splash screen on boot (optional)

To add a splash screen on boot, you can use `plymouth`. First, install the `plymouth` package and the `plymouth-theme-spinner` theme.

```bash
sudo pacman -S plymouth
```

Then, add the following line to your `/etc/mkinitcpio.conf` file to include `plymouth` in the initramfs.

```bash title="/etc/mkinitcpio.conf"
HOOKS=(... plymouth)
```

Then, add `splash` to the `GRUB_CMDLINE_LINUX_DEFAULT` variable in your `/etc/default/grub` file to enable the splash screen on boot.

```bash title="/etc/default/grub"
GRUB_CMDLINE_LINUX_DEFAULT="... splash"
```

Finally, config and regenerate the initramfs file to apply the changes.

```bash title="/etc/plymouth/plymouthd.conf"
[Daemon]
ShowDelay=0
Theme=bgrt # or any theme you like
```

```bash
sudo mkinitcpio -P
```

#### 8. Install `nvm` for Node.js version management (it's required for some Neovim plugins)

To install `nvm` (Node Version Manager) for managing Node.js versions, you can follow the steps below:

1. Download and run the `nvm` installation script:

   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
   ```

2. Restart your terminal and run the following command to install the latest LTS version of Node.js:

   ```bash
   # Install the latest LTS version of Node.js
   nvm install --lts

   # Use the latest LTS version of Node.js
   nvm use --lts
   ```

3. Verify the installation by checking the Node.js and npm versions:

   ```bash
   node -v
   # > v24.15.0
   npm -v
   # > 11.14.1
   ```

4. Install `bun` (optional):

   ```bash
   npm install -g bun@latest
   ```
