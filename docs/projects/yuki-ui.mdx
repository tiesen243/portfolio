## Introduction

**Yuki UI** is a personal side project by @tiesen243: a modern UI component library for React inspired by the flexibility and accessibility of [shadcn/ui](https://ui.shadcn.com/). This portfolio page demonstrates the project, its commands, and how the library is used in demo Next.js apps.

- Repository: https://github.com/tiesen243/yuki-ui
- Homepage / docs: https://ui.tiesen.id.vn

## Quick commands

- Install dependencies:
  - bun install
- Development (local):
  - bun dev
- Build packages:
  - bun run build
- Typecheck / lint:
  - bun run typecheck
  - bun run lint

## Project layout & architecture (high level)

- Monorepo with Bun workspaces.
- apps/: contains example/demo apps (Next.js) that consume the component library and render docs.
- tools/: shared configs and developer tooling (formatters, lint rules, doc helpers).
- Central catalogs and packageManager define consistent dependency versions across the workspace.

This setup keeps components, docs, and demo apps colocated so changes to components can be immediately validated in the example apps and documentation site.

## Scripts and workflow details

- Root scripts typically run scoped commands using Bun's --filter flag; to run a command for a single package use the same filter.
  Example: bun --filter @yuki/web dev
- There is no single global "test" script in the root; run tests per-package if present.
- Formatting and linting use oxfmt and oxlint (matching the repository's devDependencies).

## Key conventions

- Bun-first: Bun is the declared package manager (packageManager: "bun@1.3.0"). Use bun install / bun dev / bun run for consistent behavior.
- Workspace filtering: Commands target packages via Bun filters (e.g., --filter @yuki/web). When editing docs or demo apps, run the app's dev script via the filter.
- Shared tooling under tools/: prefer updating shared configs (oxc, tsconfig) there to affect all workspace packages.
- Docs are MDX-driven (Fumadocs) and can include live previews; editing docs may require running the package postinstall hooks that generate MDX assets.
